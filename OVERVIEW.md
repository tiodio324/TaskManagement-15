# Обзор проекта: Система управления задачами

## 📋 Описание проекта

Данное веб-приложение представляет собой систему управления задачами для команд разработки. Система разработана как Single Page Application (SPA) без использования `react-router`, что обеспечивает быструю навигацию между разделами без перезагрузки страницы.

**Основная цель проекта**: Автоматизация управления задачами, проектами и спринтами с возможностью отслеживания прогресса и совместной работы.

## 🎯 Основные возможности системы

### Для всех пользователей (включая неавторизованных):

1. **Просмотр задач**
   - Список всех задач
   - Фильтрация по проекту, статусу, приоритету
   - Поиск по названию и описанию

2. **Просмотр проектов**
   - Каталог проектов
   - Информация о проектах
   - Статистика задач

3. **Просмотр спринтов**
   - Список спринтов
   - Задачи в спринте
   - Прогресс выполнения

### Для авторизованных пользователей (Разработчик):

1. **Управление задачами**
   - Создание новых задач
   - Редактирование и обновление статуса
   - Добавление комментариев

2. **Работа со спринтами**
   - Просмотр своих задач в спринте
   - Обновление прогресса
   - Логирование времени

3. **Комментирование**
   - Добавление комментариев к задачам
   - Обсуждение с командой

### Для администраторов:

Все функции разработчика, плюс:

1. **Управление задачами** (CRUD операции)
   - Полный контроль над задачами
   - Назначение исполнителей
   - Установка приоритетов

2. **Управление проектами** (CRUD операции)
   - Создание и редактирование проектов
   - Управление участниками

3. **Управление спринтами** (CRUD операции)
   - Планирование спринтов
   - Распределение задач
   - Закрытие спринтов

## 💡 Почему React, а не Laravel (PHP)?

Проект реализован на React вместо традиционного стека Laravel + PHP по двум ключевым причинам:

1. **Встроенная безопасность после сборки (обфускация кода)**
   При выполнении команды `npm run build` весь исходный код проекта проходит через процесс **минификации и обфускации**. Это означает, что итоговые JavaScript-файлы становятся практически нечитаемыми для человека — переменные переименовываются в короткие бессмысленные имена, убираются все комментарии, код сжимается. В результате, даже если кто-то получит доступ к файлам на хостинге, восстановить исходную логику приложения будет крайне затруднительно. В Laravel исходный PHP-код остаётся на сервере в читаемом виде.

2. **Бесплатный хостинг на статичных платформах**
   После сборки React-приложение превращается в набор обычных статичных файлов (HTML, CSS, JS), которые можно разместить на **любом бесплатном статичном хостинге**, например:
   - **Vercel** — деплой в один клик, автоматический CI/CD
   - **GitHub Pages** — бесплатный хостинг прямо из репозитория

   Laravel и PHP требуют серверный хостинг с поддержкой PHP и базы данных, что практически всегда **платно**, либо доступно только с жёсткими ограничениями и сложной настройкой.

## 🔐 Система ролей и прав доступа

### Роли пользователей:

1. **viewer** (Наблюдатель) - роль по умолчанию
   - Просмотр задач, проектов, спринтов
   - Нет возможности редактирования

2. **developer** (Разработчик) - требует авторизацию

3. **admin** (Администратор) - требует авторизацию

   - Полный доступ ко всем данным
   - Доступ к разделу "Администрирование"

**Пароли для входа** находятся в файле `CREDENTIALS.md`

## 🔍 Краткий экскурс в проект для защиты

Если вы знакомы с HTML, CSS и JavaScript, но не работали с React и TypeScript — этот раздел поможет вам быстро понять, как устроен проект и что где находится.

### Что такое React?

React — это JavaScript-библиотека для создания пользовательских интерфейсов. Вместо обычного HTML здесь используется **JSX** (JavaScript XML) — расширение синтаксиса JS, которое позволяет писать разметку прямо в JavaScript-коде. Файлы с расширением `.tsx` — это то же самое, но с поддержкой TypeScript.

**Простая аналогия**: если в обычном веб-сайте у вас есть `index.html` + `script.js`, то в React весь HTML и JS объединены в **компоненты** — функции, которые возвращают разметку.

### Что такое TypeScript?

TypeScript — это JavaScript с **типизацией**. Он добавляет возможность указывать типы переменных, параметров функций и возвращаемых значений. Это помогает находить ошибки ещё до запуска кода. После сборки TypeScript превращается в обычный JavaScript.

**Простая аналогия**: TypeScript — это как проверка орфографии для кода. Файлы `types/*.types.ts` описывают «форму» данных — какие поля есть у объекта и какого они типа. Это похоже на описание колонок в таблице базы данных, но на уровне кода.

### Как устроен проект?

#### 1. Точка входа — `main.tsx` и `App.tsx`

Всё начинается с файла `main.tsx`, который рендерит (отображает) компонент `App` на странице. `App.tsx` — это главный компонент приложения. При загрузке он вызывает `dataStore.loadAllData()` — загружает все данные из Firebase, оборачивает содержимое в `MainLayout` (шапка + боковое меню + контент) и содержит `PageRouter` — компонент, который решает, какую страницу показать.

#### 2. Навигация (без react-router)

В проекте **нет** библиотеки `react-router`. Навигация реализована вручную через **MobX store**:

- `NavigationStore` хранит переменную `currentPage` (например, `'home'`, `'tasks'`, `'projects'`, `'sprints'`, `'admin'`)
- Компонент `PageRouter` в `App.tsx` через `switch/case` проверяет `currentPage` и рендерит нужную страницу:
  - `'home'` → `HomePage` (дашборд)
  - `'tasks'` → `TasksPage` (задачи)
  - `'projects'` → `ProjectsPage` (проекты)
  - `'sprints'` → `SprintsPage` (спринты)
  - `'admin'` → `AdminPage` (администрирование)
- Когда пользователь нажимает на пункт в боковом меню (`Sidebar`), вызывается `navigationStore.navigate('tasks')`, что меняет `currentPage`, и React автоматически перерисовывает нужную страницу

#### 3. Управление состоянием — MobX

**MobX** — это библиотека для управления состоянием приложения (аналог «глобальных переменных», доступных из любого компонента). В проекте 4 store:

- **AuthStore** (`store/AuthStore.ts`) — авторизация и роли пользователей
- **DataStore** (`store/DataStore.ts`) — все данные приложения (задачи, проекты, спринты, комментарии) и CRUD-операции
- **NavigationStore** (`store/NavigationStore.ts`) — текущая страница и навигация
- **UIStore** (`store/UIStore.ts`) — управление модальными окнами, уведомлениями, темой

Компоненты оборачиваются в `observer()` из `mobx-react-lite` — это позволяет React автоматически перерисовывать компонент при изменении данных в store.

#### 4. Авторизация и роли

`AuthStore` управляет системой ролей:

- **viewer** (Наблюдатель) — роль по умолчанию, только просмотр задач, проектов и спринтов
- **developer** (Разработчик) — создание и редактирование задач, комментирование
- **admin** (Администратор) — полный доступ: управление проектами и спринтами, назначение исполнителей
- Пароли захардкожены в коде (это учебный проект, не production)
- Сессия сохраняется в `localStorage` на 24 часа
- Проверка прав: объект `ROLE_PERMISSIONS` определяет, что может делать каждая роль
- При попытке перейти на защищённую страницу без авторизации открывается модальное окно входа

#### 5. Работа с данными — Firebase

`firebase.ts` инициализирует подключение к **Firebase Realtime Database** (облачная NoSQL база данных от Google). Класс `FirebaseService` предоставляет методы:

- `getData(path)` — получить данные
- `setData(path, data)` — записать данные
- `updateData(path, updates)` — обновить данные

`DataStore` использует эти методы для CRUD-операций с задачами, проектами, спринтами и комментариями. Каждая сущность хранится в массиве, при создании/редактировании данные сначала отправляются в Firebase, а затем обновляются локально.

#### 6. Компоненты и страницы

**Компоненты** (`src/components/`):
- `UI/` — переиспользуемые элементы: Button, Input, Modal, Table, Select, Badge, Card
- `Layout/` — структура страницы: Header (шапка), Sidebar (боковое меню), MainLayout (общий каркас)
- `LoginModal/` — модальное окно авторизации
- `ConfirmModal/` — модальное окно подтверждения действий (например, удаления)

**Страницы** (`src/pages/`):
- `HomePage/` — дашборд со статистикой: количество задач по статусам, активные проекты и спринты
- `TasksPage/` — список задач с фильтрацией по проекту, статусу (To Do/In Progress/Review/Done) и приоритету
- `ProjectsPage/` — каталог проектов с информацией об участниках и статистикой задач
- `SprintsPage/` — управление спринтами: планирование, распределение задач, отслеживание прогресса
- `AdminPage/` — панель администрирования с CRUD-операциями для всех сущностей

Каждая страница — это папка с тремя файлами:
- `[PageName].tsx` — React-компонент с логикой и разметкой
- `[PageName].module.scss` — стили (CSS-модули + SCSS)
- `index.ts` — реэкспорт компонента для удобного импорта

#### 7. Стили

Проект использует **SCSS** (расширение CSS) с модульным подходом:
- `*.module.scss` — стили, привязанные к конкретному компоненту (не конфликтуют с другими)
- `styles/globalColors.scss` — цветовая палитра проекта (продуктивная сине-оранжевая тема)
- `styles/variables.scss` — CSS-переменные (размеры, отступы, шрифты)
- `styles/mixins.scss` — повторно используемые блоки стилей
- `styles/global.scss` — глобальные стили для всего приложения
